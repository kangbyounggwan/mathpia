# 리뷰 피드백 통합 노트

## 통합하는 피드백

### 1. Gemini AI 응답 캐싱 전략 추가
**이유**: AI 분석 결과를 매번 호출하면 비용과 지연이 심각. 캐싱 필수.
**적용**: analyticsStore에 캐시 TTL 로직 추가. 새 풀이 데이터 5건 이상일 때만 재분석.

### 2. Gemini API 응답 검증 + 재시도
**이유**: AI 응답이 JSON 형식을 따르지 않을 수 있음. 기존에도 이 문제가 있었음.
**적용**: 모든 Gemini 서비스에 JSON Schema 검증 + 최대 2회 재시도 + 폴백 값 설정.

### 3. 선생님 탭 구조 개선
**이유**: 7탭은 과도함. 대시보드 내 진입점이 더 자연스러움.
**적용**: 문제은행과 학생분석은 대시보드에서 Stack 네비게이션으로 진입. 기존 5탭 유지.

### 4. 차트 라이브러리 전략 명확화
**이유**: 커스텀 SVG 차트는 구현 복잡도 높음.
**적용**: 레이더 차트만 react-native-svg 커스텀. 라인/바 차트는 간단한 커스텀 또는 라이브러리 활용.

### 5. 서비스 팩토리 패턴 추가
**이유**: 서비스 인스턴스 관리 방법이 명시되지 않았음.
**적용**: src/services/index.ts에 서비스 팩토리 패턴 추가.

### 6. AI 분석 트리거 시점 명확화
**이유**: 언제 AI 분석을 실행하는지 모호했음.
**적용**: 학생-화면 진입 시 캐시 확인, 선생님-수동 버튼, 학부모-화면 진입 시 캐시.

### 7. 로딩/에러 UX
**이유**: Gemini API 3-10초 소요. 사용자 경험 필수.
**적용**: 스켈레톤 UI + 진행 메시지 + Error Boundary.

### 8. 오답노트 숙련 기준 명확화
**이유**: "3회 연속 정답" 기준이 모호.
**적용**: 최소 24시간 간격의 3회 연속 정답 → "숙련" 마크.

### 9. 학생 목록에서 분석 진입
**이유**: students.tsx에서 학생 클릭 → 분석으로 이동이 자연스러운 UX.
**적용**: 학생 카드에 "분석" 버튼 또는 카드 클릭으로 student-analytics 진입.

### 10. Mock 문제 100개+
**이유**: 50개는 단원별 1-2문제밖에 안됨. 분석에 불충분.
**적용**: 최소 100문제, 주요 단원별 5문제 이상으로 확대.

## 통합하지 않는 피드백

### 1. AsyncStorage 대신 expo-file-system 병행
**이유**: 프론트엔드 프로토타입 단계에서 과도한 복잡성. AsyncStorage 6MB 제한은 문제 100개 + 분석 데이터로는 충분함. Supabase 전환 시 어차피 로컬 스토리지 필요 없어짐.

### 2. 학부모-자녀 연결 권한 검증
**이유**: Mock 데이터에서는 의미 없음. 서비스 인터페이스에 userId 파라미터는 이미 포함될 예정이므로 Supabase 전환 시 RLS로 자연스럽게 해결.

### 3. 다국어/접근성
**이유**: 이번 고도화 범위 외. 한국 학원 전용 앱으로 한국어만 필요. 접근성은 중요하지만 별도 태스크로.

### 4. 오프라인 상태 감지
**이유**: 프론트엔드 프로토타입에서 오프라인 지원은 과도함. Supabase 연동 시 함께 구현.
