# Mathpia UI 전면 고도화 스펙

## 프로젝트 개요
Mathpia는 한국 수학 학원용 태블릿 앱(Expo SDK 54, React Native, TypeScript)으로, 선생님/학생/학부모 3개 역할 기반 서비스이다. 현재 기능 구현은 완료되었으나 UI/UX가 프로토타입 수준이므로 **상용 수준의 세련된 앱**으로 전면 고도화한다.

## 현재 기술 스택
- Expo SDK 54 / React Native 0.81.5 / TypeScript
- react-native-paper (Material Design 3)
- react-native-svg (차트)
- expo-router (파일 기반 라우팅)
- zustand v5 (상태 관리)
- react-native-reanimated (애니메이션)

## 현재 UI 상태 진단

### 잘 된 부분
- 역할별(선생님/학생/학부모) 탭 내비게이션 구조
- react-native-paper 기반 일관된 Material Design 3 적용
- 공통 컴포넌트(Button, Input, Card, MathText) 존재
- theme.ts에 색상/스페이싱/borderRadius 정의됨

### 문제점
1. **디자인 시스템 미완성**: 하드코딩된 폰트 크기/패딩 산재, 타이포그래피 체계 없음
2. **단조로운 색상**: 파란색 일색, 역할별 시각적 차별화 없음
3. **로딩 경험 부재**: 대부분 화면에서 스켈레톤/로딩 UI 없음
4. **애니메이션 없음**: reanimated 설치만 되어있고 실제 사용 없음
5. **반응형 부족**: 768px 단일 브레이크포인트, 소형 폰 미대응
6. **접근성 미비**: 포커스 상태, 스크린리더 라벨, 명암비 미검증
7. **폼 UX 부족**: 유효성 검사 피드백, 비밀번호 강도 표시 없음
8. **화면 길이**: 일부 화면 컴포넌트가 600줄 이상으로 분리 필요
9. **다크모드 미지원**: 라이트모드만 존재
10. **빈 상태(Empty State)**: 데이터 없을 때 안내 UI 없음

## 고도화 목표

### 1. 디자인 토큰 시스템 완성
- 타이포그래피 스케일 정립 (Heading1~Caption, 시맨틱 네이밍)
- 색상 팔레트 확장 (역할별 accent 색상: 선생님=인디고, 학생=블루, 학부모=그린)
- 그림자/엘리베이션 시스템 정리 (shadow.sm, shadow.md, shadow.lg)
- 아이콘 크기 표준화
- 모든 하드코딩 수치를 토큰으로 교체

### 2. 공통 컴포넌트 강화
- **SkeletonLoader**: 카드형, 리스트형, 텍스트형 스켈레톤
- **EmptyState**: 아이콘 + 메시지 + CTA 버튼 패턴
- **Toast/Snackbar**: 성공/에러/정보 알림
- **Badge**: 숫자 배지, 상태 배지 (진행중/완료/지연)
- **BottomSheet**: 필터, 상세정보용 바텀시트
- **SearchBar**: 통합 검색바 컴포넌트
- **Avatar**: 이니셜/이미지 기반 아바타
- **ProgressRing**: 원형 진행률 표시
- **StatCard**: 통계 카드 (아이콘+숫자+라벨+트렌드)
- **SectionHeader**: 섹션 제목 + "더보기" 패턴
- **Divider**: 구분선 (라벨 포함)
- **Tag/Chip**: 난이도, 과목, 상태 태그
- 기존 Button/Input/Card 컴포넌트 리팩토링 및 변형 추가

### 3. 로그인/온보딩 화면 고도화
- 앱 로고 + 브랜딩 강화 (그라데이션 배경 또는 일러스트)
- 소셜 로그인 버튼 영역 (추후 연동 대비 UI만)
- 비밀번호 강도 인디케이터
- 폼 필드 유효성 실시간 검사 + 에러 메시지 개선
- 로그인 성공 시 전환 애니메이션
- 테스트 계정 정보를 접이식(collapsible)으로 변경

### 4. 학생 화면 고도화

#### 4-1. 학생 대시보드
- 인사 헤더 ("김민수님, 오늘도 화이팅!") + 아바타
- 오늘의 학습 요약 카드 (푼 문제/정답률/연속일)
- 마감 임박 숙제 카드 (카운트다운 표시)
- 주간 학습 히트맵 (잔디밭 스타일)
- 최근 오답 문제 미리보기 슬라이더

#### 4-2. 숙제 화면
- 탭 필터 (전체/진행중/완료/지연) 개선
- 숙제 카드에 진행률 바 + 남은시간 표시
- 숙제 상세: 문제 목록 + 제출 상태 한눈에 보기

#### 4-3. 문제 풀기 화면
- 문제 번호 네비게이션 바 (풀음/안풀음 상태 표시)
- 타이머 표시
- AI 힌트 버튼 플로팅
- 풀이 캔버스 개선 (펜/지우개 크기 조절)
- 제출 확인 모달

#### 4-4. 오답노트 화면
- 마스터리 진행률 큰 표시
- 카드 뒤집기 효과 (문제↔풀이)
- 복습 모드 진입 시 카운트다운 + 집중모드 UI
- 필터 칩 개선

#### 4-5. 분석 화면
- 레이더 차트 크기/가독성 개선
- 약점 분석 카드 강조
- 주간/월간 트렌드 토글
- AI 리포트 섹션 시각적 구분

### 5. 선생님 화면 고도화

#### 5-1. 선생님 대시보드
- 오늘의 현황 요약 (제출대기/채점필요/학생수)
- 최근 활동 타임라인
- 빠른 액션 그리드 (숙제출제/문제등록/채점하기/성적분석)
- 주의 필요 학생 알림 카드

#### 5-2. 학생 관리
- 학생 목록 카드형/리스트형 토글
- 학생 카드: 아바타 + 이름 + 최근 정답률 + 상태배지
- 학생 상세: 탭(기본정보/성적/숙제/오답)
- 학생 검색 및 필터

#### 5-3. 숙제 관리
- 숙제 생성 위저드 (단계별 스텝퍼)
- 숙제 카드: 진행현황 미니 차트
- 문제 선택 UI 개선 (드래그&드롭 정렬)

#### 5-4. 채점 화면
- 학생 답안 + 캔버스 풀이 + 정답 나란히 비교
- 원터치 정답/오답 마킹
- 코멘트 입력 + 음성메모 영역(UI만)
- 일괄 채점 모드

#### 5-5. 문제은행
- 그리드/리스트 뷰 토글
- 문제 미리보기 모달 (수식 렌더링 포함)
- 필터 패널 사이드바 (웹) / 바텀시트 (모바일)
- 대량 등록 진행률 UI

#### 5-6. 학생 분석
- 개별 학생 vs 반 평균 비교 차트
- 약점 분석 → 추천 문제 연결 흐름
- PDF 리포트 내보내기 UI(버튼만)

### 6. 학부모 화면 고도화

#### 6-1. 학부모 대시보드
- 자녀 선택기 개선 (아바타 + 이름 + 오늘 학습 요약)
- 주요 지표 카드 (정답률/푼 문제/연속학습일/숙제완료율)
- 약점 과목 경고 배너
- AI 맞춤 조언 카드 (말풍선 스타일)

#### 6-2. 스케줄
- 캘린더 주간/월간 토글
- 날짜별 학습/숙제 이벤트 표시
- 마감 임박 알림 강조

#### 6-3. 리포트
- 기간 선택 (주간/월간/전체)
- 과목별 성적 추이 라인 차트
- 강점/약점 요약 카드
- 선생님 코멘트 영역

### 7. 애니메이션 & 인터랙션
- 화면 전환: 슬라이드/페이드 트랜지션 (react-native-reanimated)
- 리스트: FlatList entering/exiting 애니메이션
- 카드: 터치 시 scale 피드백 (0.98)
- 진행률 바/원: 숫자 카운트업 애니메이션
- 탭 전환: 슬라이드 인디케이터 애니메이션
- 스켈레톤: shimmer 효과
- Toast: 위에서 슬라이드인/아웃
- 차트: 데이터 로딩 시 그리기 애니메이션

### 8. 반응형 & 접근성
- 브레이크포인트: small(<375), medium(375-768), large(>768)
- useResponsive 커스텀 훅 (현재 화면 크기 기반 값 반환)
- 모든 아이콘 버튼에 accessibilityLabel
- 명암비 4.5:1 이상 보장
- 포커스 표시 상태 (키보드 내비게이션)
- 최소 터치 영역 44px 보장

### 9. 다크모드
- theme.ts에 darkColors 팔레트 추가
- useColorScheme 기반 자동 전환
- 프로필에서 수동 테마 전환 (시스템/라이트/다크)
- 모든 화면에서 다크모드 색상 적용

### 10. 에러 & 빈 상태
- ErrorBoundary 래퍼 컴포넌트
- 네트워크 에러 화면 (재시도 버튼)
- 데이터 없음 빈 상태 (아이콘 + 안내 메시지 + CTA)
- 각 화면별 맞춤 빈 상태 일러스트/메시지

## 제약 조건
- 기존 기능 로직은 수정하지 않음 (UI/스타일만 변경)
- 새 패키지 최소 설치 (가급적 기존 라이브러리 활용)
- react-native-paper 기반 유지 (MD3 활용 극대화)
- 웹(Expo Web) 호환성 유지
- TypeScript strict 모드 유지
- 기존 서비스/스토어 레이어 변경 없음

## 우선순위
1. 디자인 토큰 + 공통 컴포넌트 (기반)
2. 로그인/온보딩 (첫인상)
3. 학생 화면 (주요 사용자)
4. 선생님 화면 (핵심 관리자)
5. 학부모 화면
6. 애니메이션 & 인터랙션
7. 다크모드
8. 접근성 최종 점검
