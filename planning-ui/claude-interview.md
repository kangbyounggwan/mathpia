# Mathpia UI 고도화 인터뷰 기록

## Q1: 앱의 주 타겟 디바이스는?
**A: 태블릿 우선** - 태블릿(10인치+)이 주력, 모바일/웹도 지원

## Q2: 게이미피케이션 요소 도입 여부?
**A: 기본만** - 스트릭(연속학습일) + 진행률 애니메이션 정도만

## Q3: 다크모드 포함 여부?
**A: 필요 없음** - 라이트 모드만 유지

## Q4: 한글 커스텀 폰트?
**A: Noto Sans KR** - Google 폰트, 다국어 지원 우수

## Q5: 역할별 색상 차별화?
**A: 역할별 액센트** - 선생님=인디고, 학생=블루, 학부모=그린 액센트 색상

## Q6: 로그인 화면 스타일?
**A: 미니말 클린** - 현재와 비슷하되 디테일 개선

## Q7: 가장 중요한 고도화 영역 (우선순위)?
**A: 디자인 시스템 우선** - 토큰/컴포넌트 기반을 먼저 완성하고 화면에 적용

## Q8: 애니메이션 수준?
**A: 최소한** - 스켈레톤 로더와 기본 전환 애니메이션만

## Q9: 새로운 공통 컴포넌트 중 가장 필요한 것?
**A: SkeletonLoader + EmptyState** - 로딩 스켈레톤과 빈 상태 UI (가장 시급)

## 인터뷰 요약

### 핵심 방향
- **태블릿 퍼스트**, 모바일/웹 호환
- **Noto Sans KR** 커스텀 폰트 적용
- **역할별 액센트 색상** 차별화 (인디고/블루/그린)
- **다크모드 제외**, 라이트 모드만
- **최소 애니메이션** (스켈레톤 + 기본 전환)
- **디자인 토큰 시스템이 1순위** 기반 작업
- **SkeletonLoader + EmptyState** 공통 컴포넌트 우선
- **게이미피케이션은 기본 수준** (스트릭 + 진행률)
- **로그인은 미니말 클린** 스타일
