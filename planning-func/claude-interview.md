# Interview Transcript - Mathpia 기능 고도화

## Q1: 핵심 플로우 우선순위
**Q**: 1단계에서 가장 먼저 구현하고 싶은 핵심 플로우는?
**A**: **Gemini AI 기능 우선**

## Q2: 백엔드 선택
**Q**: 백엔드를 Supabase로 확정했나요?
**A**: **Supabase 확정** (Auth + PostgreSQL + Storage)

## Q3: Gemini AI 우선순위
**Q**: Gemini AI 연동의 우선순위는?
**A**: **PDF 문제 추출이 최우선**

## Q4: 일정
**Q**: 앱의 실제 사용 시점은?
**A**: **1-2주 내 데모/프로토타입**

## Q5: Supabase 상태
**Q**: Supabase 프로젝트를 이미 생성했나요?
**A**: **아직 안 만듦** - 계획에 Supabase 셋업도 포함해야 함

## Q6: 데모 필수 기능
**Q**: 1-2주 데모에서 반드시 작동해야 하는 기능은?
**A**: **전부 다:**
- PDF → 문제 추출 + 문제은행 저장
- 과제 → 풀이 → 채점 전체 플로우
- 실제 로그인 + 데이터 저장

## Q7: PDF 추출 범위
**Q**: PDF 문제 추출 시 이미지 내 수식도 인식해야 하나요?
**A**: **사진 촬영도 지원** - 태블릿 카메라로 교재 촬영 → 문제 추출까지

## 종합 요약

### 우선순위 (1-2주 데모 목표):
1. **Supabase 셋업** - 프로젝트 생성, 스키마, Auth
2. **실제 인증** - Supabase Auth (이메일/비밀번호)
3. **PDF/사진 → AI 문제 추출** - Gemini Vision으로 이미지 인식 포함
4. **과제 생성 → 풀이 → 채점 전체 플로우** - 실제 데이터 저장
5. **화면-스토어 연결** - 모든 하드코딩 데이터 제거

### 핵심 결정사항:
- 백엔드: Supabase (확정)
- AI: Gemini 2.5 Flash (확정)
- PDF 추출: 이미지/사진 포함 (Gemini Vision)
- 일정: 1-2주 (매우 빠듯함)
